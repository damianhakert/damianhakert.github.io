GitLab CI (Continuous Integration) integrates with your GitLab installation to run tests for your projects.
Login with a GitLab account, add a project with one click and enjoy running your tests.
You can install and use on your servers or run it for free on ci.gitlab.com

Source code

Advantages

  • GitLab CI is easy to set up, it is included with the Omnibus packages of GitLab CE and EE
  • Adding a project to it and setting up the hooks in the repository is done only takes one click
  • It has a nice interface with a clear menu structure and easy to understand navigation
  • You can see the test results inside the merge request of GitLab
  • From the merge request it is a single click to see the realtime build log output
  • The tests run distributed on separate machines, so that the CI server is performant and stable
  • One commit can have on multiple jobs, by running in parallel you get the test restult faster
  • It is free to use, open source and a new features come out every month

Requirements

GitLab CI is officially supported on these Linux distributions:

  • Ubuntu
  • Debian/GNU
  • CentOS
  • Red Hat Enterprise Linux (please use the CentOS packages and instructions)
  • Scientific Linux (please use the CentOS packages and instructions)
  • Oracle Linux (please use the CentOS packages and instructions)
