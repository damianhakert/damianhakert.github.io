Upgrading from a source installation to Omnibus

If you are using an old source installation of GitLab CE, you can follow this migration path to bring your installation up-to-date:

  1. Upgrade your source installation up to at least version 6.6. Please consider reaching out to us for assistance;
  2. On a fresh server, install the Omnibus package with the same version (including the minor and patch version) of your now-upgraded source installation;
  3. Backup the source installation using the backup/restore raketask ;
  4. Restore the backup on the new Omnibus installation;
  5. If needed, upgrade the Omnibus installation to the latest version.

Migrating from SVN to GitLab

Migrating from SVN to GitLab also involves moving to another workflow, the git flow.

  1. Free resources for learning the new workflow include the GitLab flow and the Platzi online tutorial;
  2. Install and setup GitLab using the Omnibus package;
  3. Follow the instructions for migrating repositories from SVN to GitLab .
