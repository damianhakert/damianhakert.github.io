Installing GitLab

Below are all the options to create a GitLab installation. If you're not sure which method to choose, please use the package installation method.

Omnibus Package installation (recommended)

We recommend installing GitLab in 2 minutes using our packages (deb/rpm). A package contains GitLab and all its depencies (Ruby, PostgreSQL, Redis, Nginx, Unicorn, etc.). For troubleshooting and configuration options please see the Omnibus GitLab readme.

  • Gitlab Community Edition packages

  • Gitlab Enterprise Edition packages (subscribers only)

Other official installation methods

  • GitLab Omnibus Cookbook can be used for production installations

  • Manual installation guide set up a production server on Ubuntu offers detailed and complete step-by-step instructions

Third party one-click installers

  • Digital Ocean 1-Click Application Install gets a new server up in 55 seconds with fast SSD disks, we recommend selecting a droplet with 2GB of memory, their blog post contains a promo code for two months free and there is a nice tutorial

  • Google Compute Engine offers a click to deploy GitLab template

  • BitNami one-click installers contains both GitLab and GitLab CI and is available as installer, virtual machine or for cloud hosting providers (Amazon Web Services/Azure/etc.)

  • Cloud 66 deployment and management to deploy GitLab to your own server or any cloud (DigitalOcean/AWS/Rackspace/GCE/etc.) and then manage it with database backups, scaling and more

  • Packager.io provides autogenerated debian or rpm packages of GitLab for Debian, Ubuntu, and CentOS/RHEL

  • twoub lets you deploy GitLab on their cloud service hosted in Germany

Community installation methods

  • GitLab recipes repository with unofficial guides for using GitLab with different software (operating systems, webservers, etc.) than the official version

  • Unofficial installation guides on the public wiki to install GitLab on different operating systems

  • Docker Sameer Naik made a nice Dockerfile to build a GitLab container image

  • Puppet has multiple modules, Sebastien Badia made a popular one and Spencer Owen made one that work with 7.0 and up

  • Vagrant option by Tuomo Tanskanen (also available for GitLab CI)

GitLab Development installation

If you want to contribute to GitLab we suggest you select the GitLab Development Kit to install the testing dependencies since it is much faster and more reliable than the cookbook options.

  • Gitlab Development Kit (recommended) lets you install GitLab isolated in a directory with a single make command

  • Development installation option in the GitLab Cookbook readme uses the cookbook to install on the host machine or on a VM with Vagrant

GitLab as a Service - Hosted GitLab

  • GitLab.com free, hosted GitLab by GitLab B.V., also the canonical location of the GitLab source code

  • GitHost.io private, managed GitLab instances in the Cloud

  • Geeklist is a social platform for developers

  • Cloudhost.io offers a multi tenant service

Third-party applications

  • There are several third-party applications that support GitLab.
