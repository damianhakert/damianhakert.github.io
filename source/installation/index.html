GitLab Installation

Below are all the options to create a GitLab installation. If you're not sure which method to choose, please use the Omnibus package installation method.

Omnibus package installation (recommended)

We recommend installing the Omnibus package instead of installing GitLab from source. Omnibus GitLab takes just 2 minutes to install and is packaged in the popular deb and rpm formats. Compared to an installation from source, the Omnibus package is faster to install and upgrade, more reliable to upgrade and maintain, and it shortens the response time for our subscribers' issues. A package contains GitLab and all its depencies (Ruby, PostgreSQL, Redis, Nginx, Unicorn, etc.). For troubleshooting and configuration options please see the Omnibus GitLab readme.

Other official installation methods

  • GitLab Development Kit lets you install GitLab for development purposes, it is isolated in a directory and runs with a single make command

  • GitLab Omnibus Cookbook can be used for production Chef installations

  • Installation from source guide to set up a production server on Ubuntu with detailed and complete step-by-step instructions

  • Docker files to create a production image based on the Omnibus packages

  • Omnibus package for Raspberry Pi 2 with Raspbian OS (not the latest version, package gets updated periodically)

One-click install providers

One-click installers are frequently out of date and might not contain our Omnibus packages. The worst example of these are the Bitnami packages that can't be updated at all. We advise to not use one-click installers but instead start an vanilla Ubuntu 14.04 instance and use the recommended Omnibus package installation. This is almost as quick as a one-click install and you're sure of the latest version and easy upgrades.

  • GitLab Community Edition AWS AMI

  • Digital Ocean 1-Click Application Install gets a new server up in 55 seconds with fast SSD disks. We recommend selecting a droplet with 2GB of memory. Check their blog post and a nice tutorial

  • Google Compute Engine offers a click to deploy GitLab template

  • Cloud 66 deployment and management to deploy GitLab to your own server or any cloud (DigitalOcean/AWS/Rackspace/GCE/etc.) and then manage it with database backups, scaling and more

  • Packager.io provides autogenerated debian or rpm packages of GitLab for Debian, Ubuntu, and CentOS/RHEL

  • twoub lets you deploy GitLab on their cloud service hosted in Germany

Community installation methods

  • GitLab recipes repository with unofficial guides for using GitLab with different software (operating systems, webservers, etc.) than the official version

  • Unofficial installation guides on the public wiki to install GitLab on different operating systems

  • Docker image made by Sameer Naik based on the source installation method

  • Puppet has multiple modules, Sebastien Badia made a popular one and Spencer Owen made one that work with 7.0 and up

  • Vagrant option by Tuomo Tanskanen (also available for GitLab CI)
