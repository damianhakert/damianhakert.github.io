Features

GitLab offers everything you expect from a modern code collaboration tool. Below is a short list, for a visual overview please see the screenshots on GitLab.org

  • good-looking diff views
  • merge requests for code review
  • line specific comments
  • issues with attachments
  • issue labels and assignments
  • issue milestones / sprints
  • link to an external issue tracker
  • activity feed per project
  • user dashboard with filters
  • a wiki which is stored in git
  • project wall
  • code snippets
  • branches and a network graph
  • unique permission levels per user and project (guest, reporter, developer, master, owner)
  • protected branches (only masters can push to protected branches)
  • forking of repo's
  • teams consisting of multiple people
  • groups consisting of multiple projects
  • multiple deploy keys are possible for the same project
  • multiple projects are possible for the same deploy key
  • global and per project notification settings
  • http(s) and ssh git access
  • LDAP support
  • web hooks
  • convenient backups
  • administrative interface
  • web based editor
  • code search
  • runs on physical and virtual machines

Features
Gitlab.org is quickly becoming popular because it is easier to install and has all the important features such as merge requests. We think that the workflow enabled by merge requests is making software development a lot more fun.

Community
Gitlab.org is open source software and can be used and modified free of charge. It is actively maintained, has an active community and is in active use within many companies such as Thomson Reuters. Gitlab is in the top 50 most popular open source projects.

Git
Gitlab is based on the Git version control system. A good reference is the free Pro Git ebook and Clay Shirky gave a good presentation about the potential of decentralized version control to transform society.
